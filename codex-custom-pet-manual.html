1. カスタムペットとは EXPERIMENTAL

Codex のターミナル UI (TUI) と Desktop App には、 デスクトップ上に小さなマスコットキャラクターを常駐させる 「ペット」機能がある。 1990 年代の Office アシスタント (イルカのカイルなど) を AI 時代に 現代化したような機能で、 タスク状態 (アイドル / 実行中 / 待機 / 失敗) に応じて 自動でアニメーションが切り替わる。

既定では 8 種の bundled キャラクター (Codex / Dewey / Fireball / Rocky / Seedy / Stacky / BSOD / Null Signal) が用意されているが、 ~/.codex/pets/ に自前のスプライトと設定ファイルを置けば、 完全オリジナルのマスコットを動かせる。

Desktop App での起動コマンドは日本語ローカライズで /なでる、 英語環境では /pet。 入力欄にこれを打って Enter すると右下にペットが出現する (toggle)。

2. 動作環境

環境対応備考
Codex Desktop App ✅ 推奨 v26 系 Electron アプリ。 macOS/Windows 対応。 GUI で pet 一覧 + 選択 UI 完備
Codex TUI (CLI) ⏳ 0.129-alpha PR #21206 (tui-pets) で実装、 stable は merge 待ち。 ターミナル内に sprite 重畳描画
Codex CLI v0.128 stable ❌ TUI ペット未実装 config.toml に書いても無視。 pet.json + spritesheet は配置しても OK (将来対応)

カスタムペットファイルの配置場所自体は全環境で共通 ($CODEX_HOME/pets/)。

3. Spritesheet 仕様

ファイル形式WebP RGBA (透明背景必須)。 RGB は認識されないことがある
合計サイズ1,536 × 1,872 px
Frame 単位192 × 208 px
グリッド8 列 × 9 行 = 72 frames
キャラの占有frame 全体ではなく 中央に約 130 × 140 px 程度に小さく配置 (周囲は透明)。 ペット感が出る
圧縮lossless 推奨 (キャラのエッジが綺麗に)、 lossy quality 85+ でも実用上問題なし

グリッド構造

横 8 列 × 縦 9 行のアトラス。 各 row が 1 アニメーションに対応。 frame 番号は 左上 = 0、 右に進んで row 末で改行 (8, 9, 10, ... の順)。

    col0  col1  col2  col3  col4  col5  col6  col7
row0  [ 0] [ 1] [ 2] [ 3] [ 4] [ 5] [ 6] [ 7]   ← idle
row1  [ 8] [ 9] [10] [11] [12] [13] [14] [15]   ← running-right
row2  [16] [17] [18] [19] [20] [21] [22] [23]   ← running-left
row3  [24] [25] [26] [27]                       ← waving (4 frames)
row4  [32] [33] [34] [35] [36]                  ← jumping (5 frames)
row5  [40] [41] [42] [43] [44] [45] [46] [47]   ← failed
row6  [48] [49] [50] [51] [52] [53]             ← waiting (6 frames)
row7  [56] [57] [58] [59] [60] [61]             ← running (6 frames)
row8  [64] [65] [66] [67] [68] [69]             ← review (6 frames)
透明背景は必須級。 過去テストで RGB のままだとペット一覧から消える事象あり。 Pillow で保存するときは format='WEBP', lossless=True + RGBA mode 維持を確認。

4. 9 種のアニメーション

各 row が 1 つのアニメーションに対応。 デフォルトの定義は以下:

Rowアニメ名用途FramesfpsLoop状況
0idle立ち姿 / 待機88true常時
1running-right右へ走る89true移動時
2running-left左へ走る89true移動時
3waving手を振る47false挨拶 / 完了
4jumpingジャンプ57false強調 / 喜び
5failedしょんぼり87falseエラー時
6waiting待機67true入力待ち
7running走る (定位置)68true推論中
8review考える67truediff レビュー時

エイリアス (任意)

呼び出しコードによっては別名を期待するため、 alias 定義もしておくと安全:

  • move_right → running-right
  • move_left → running-left
  • wave → waving
  • bounce → jumping
  • sad → failed

5. pet.json スキーマ

{
  "id": "mypet",
  "displayName": "私のペット",
  "description": "自分専用のデスクトップマスコット",
  "spritesheetPath": "spritesheet.webp",
  "frame": {
    "width": 192,
    "height": 208,
    "columns": 8,
    "rows": 9
  },
  "animations": {
    "idle": {
      "frames": [0, 1, 2, 3, 4, 5, 6, 7],
      "fps": 8,
      "loop": true,
      "fallback": ""
    },
    "running-right": {
      "frames": [8, 9, 10, 11, 12, 13, 14, 15],
      "fps": 9,
      "loop": true,
      "fallback": "idle"
    },
    "running-left":  { "frames": [16,17,18,19,20,21,22,23], "fps": 9, "loop": true,  "fallback": "idle" },
    "waving":        { "frames": [24,25,26,27],             "fps": 7, "loop": false, "fallback": "idle" },
    "jumping":       { "frames": [32,33,34,35,36],          "fps": 7, "loop": false, "fallback": "idle" },
    "failed":        { "frames": [40,41,42,43,44,45,46,47], "fps": 7, "loop": false, "fallback": "idle" },
    "waiting":       { "frames": [48,49,50,51,52,53],       "fps": 7, "loop": true,  "fallback": "idle" },
    "running":       { "frames": [56,57,58,59,60,61],       "fps": 8, "loop": true,  "fallback": "idle" },
    "review":        { "frames": [64,65,66,67,68,69],       "fps": 7, "loop": true,  "fallback": "idle" }
  },
  "aliases": {
    "move_right": "running-right",
    "move_left":  "running-left",
    "wave":       "waving",
    "bounce":     "jumping",
    "sad":        "failed"
  }
}

フィールド一覧

キー必須説明
id✅内部識別子。 半角英数 / ハイフン推奨
displayName✅UI に表示される名前 (日本語可)
description—ペット選択 UI に出る短い説明
spritesheetPath✅pet.json と同じディレクトリからの相対パス。 通常 spritesheet.webp
frame✅1 frame の width/height + grid の columns/rows
animations✅9 種のアニメーション定義 (上記)
aliases—別名マッピング (互換性のため)

6. 配置場所

~/.codex/pets/<pet-id>/
    ├── pet.json
    └── spritesheet.webp

$CODEX_HOME が設定されていない場合は ~/.codex/ がデフォルト。 <pet-id> は pet.json の id と一致させる。

config.toml で選択 (TUI 用)

[tui]
pet = "mypet"
pet_anchor = "composer"  # または "screen-bottom"

Desktop App の場合は GUI 経由で選択 (Settings → 外観 → ペット → 「mypet」 を選択)。

7. 作成方法 3 ルート

(A) Codex に依頼する 最も楽

Codex Desktop App / CLI に「キャラクターのコンセプトを伝えて pet を作って」 と依頼すると、 内蔵の image 生成ツールで sprite を作って ~/.codex/pets/<id>/ に配置するところまで自動でやってくれる。

プロンプト例

~/.codex/pets/mypet/ にカスタム Codex pet を作成してください。

【スキーマ】
- spritesheet.webp: 1536×1872 px (192×208 frame × 8 cols × 9 rows = 72 frames)
- WebP RGBA (透明背景必須)
- pet.json: id, displayName, description, frame, animations 9 種, aliases 5 種
- 各 frame は中央に 130×140 px 程度のキャラ + 周囲透明

【キャラクター設計】
<ここにコンセプトを書く: 例「青い帽子をかぶった黒猫の chibi 風」>

完了後、 配置ファイルパス + 使った画像生成手法を出力してください。
Codex のサンドボックスから ~/.codex/pets/ へ書き込めない場合は Operation not permitted で失敗する。 その場合は staged フォルダ (~/Desktop/.tmp/...) にファイルが残るので、 ターミナルから cp で手動コピーすれば良い。

(B) 自前で画像生成 + Pillow で atlas 構築

  1. キャラクターを画像生成 (DALL-E / Midjourney / Stable Diffusion / nano-banana など) で 9 ポーズ作る、 または 1 ポーズだけ作る
  2. Pillow で 1 frame を 192 × 208 にリサイズ + 周囲透明化
  3. 1,536 × 1,872 の透明 RGBA Image を作成し、 各 row × col 位置に frame を貼り付け
  4. image.save('spritesheet.webp', format='WEBP', lossless=True) で保存
  5. pet.json を上のスキーマで作成
  6. ~/.codex/pets/<id>/ に 2 ファイルを配置
from PIL import Image

# 1 frame の素材 (透明背景 PNG) を 9 種読み込む
frames = [Image.open(f'pose-{name}.png').convert('RGBA') for name in [
    'idle', 'run-r', 'run-l', 'wave', 'jump',
    'fail', 'wait', 'running', 'review'
]]
# 各 frame を 130×140 にリサイズして 192×208 中央に配置
def make_cell(src):
    cell = Image.new('RGBA', (192, 208), (0,0,0,0))
    inner = src.resize((130, 140), Image.LANCZOS)
    cell.paste(inner, ((192-130)//2, 208-140-6), inner)
    return cell

sheet = Image.new('RGBA', (1536, 1872), (0,0,0,0))
for row, frame in enumerate(frames):
    cell = make_cell(frame)
    for col in range(8):
        sheet.paste(cell, (col*192, row*208), cell)

sheet.save('spritesheet.webp', format='WEBP', lossless=True)

(C) bundled sprite を改造

既存の Codex bundled キャラ (Codex / Dewey 等) を base に色味を変えるだけでも違う印象になる。 GitHub の openai/codex リポジトリ codex-rs/tui/pets/assets/ 配下に *-spritesheet-v3.webp があるのでこれを参考に。

8. 動作確認

Codex Desktop App

  1. Codex Desktop App を起動 (既起動なら一度 quit + 再起動でリロード)
  2. 左下「設定」 → 「外観」 → 下スクロール → 「ペット」 アコーディオン展開
  3. 「カスタムペット」 セクションに自分のペットが表示されるか確認 → 「選択」 ボタン押す
  4. 「アプリに戻る」 で Settings 閉じる
  5. 入力欄に / 入力 → 候補から「なでる」 (英語版は「pet」) 選択 → Enter
  6. 画面右下にペットが出現すれば成功

Codex CLI / TUI (0.129-alpha 以降)

  1. ~/.codex/config.toml に [tui] セクションで pet = "<id>" を設定
  2. ターミナルで codex 起動 (interactive mode)
  3. composer 周辺に sprite が描画されていれば成功

9. トラブルシュート

ペット一覧に出ない

  • spritesheet が RGB で保存されている → Pillow で convert('RGBA') してから保存
  • pet.json の id とディレクトリ名が一致していない → 一致させる
  • 配置後に Codex Desktop が起動中 → quit + 再起動でリロード

ペットが起動しても真っ黒

  • spritesheet の各 frame が透明ではなく黒で塗られている → 周囲を (0,0,0,0) 透明 RGBA に
  • frame サイズが 192×208 でない → Pillow で確認 + リサイズ

Codex 経由で配置失敗 (Operation not permitted)

  • Codex のサンドボックス制限。 staged ファイルは ~/Desktop/.tmp/<hatch-名>/package/<id>/ に残る
  • ターミナルから手動 cp でコピー: 
    cp ~/Desktop/.tmp/hatch-pet-mypet/package/mypet/pet.json ~/.codex/pets/mypet/
cp ~/Desktop/.tmp/hatch-pet-mypet/package/mypet/spritesheet.webp ~/.codex/pets/mypet/

アニメーションが切り替わらない

  • row 番号と frame index が合っていない (例: row3 の waving が [24,25,26,27] ではなく [3,4,5,6] になってる) → 列優先ではなく 行優先。 row N の最初の frame は N × 8
  • fps が大きすぎる / 小さすぎる → 7〜10 fps が目安
