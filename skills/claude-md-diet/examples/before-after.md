### 問題点

- **1800 行 = セッション開始時 ~30K トークン消費**（週次の 5% に相当）
- 完了セクションが **1000 行以上** 占有、古い情報も混在
- 機材情報が 80 行、マシン追加のたびに肥大化
- ルール類が散在、「操作上の注意」と「コーディング規約」と「情報の保存方針」が重複気味
- 新しいセッションで AI が何を優先していいかわからない

### 改善

- **CLAUDE.md = 250 行 = ~4K トークン**（1/8 に削減）
- **セッション開始時に読まれるのは CLAUDE.md のみ**、他ファイルは必要時にロード
- ドメイン別に分離、新しい情報を追加する場所が明確
- 複数 Claude Code インスタンスで並列編集しても衝突しにくい
- 月次 archive で古いものが自動整理される（claude-md-diet 月次運用）

## セッション開始時の体感差

### Before
- セッション開始 → CLAUDE.md 1800 行読み込み → 3-5 秒の処理時間 → AI が「何が重要か」推測モード
- 週次トークン 5% 以上をセッション開始だけで消費
- 古い完了セクションまで読んで「え、あの作業やり直す？」みたいな勘違いが発生

### After
- セッション開始 → CLAUDE.md 250 行読み込み → 瞬時に現状把握 → ユーザー指示に集中
- 週次トークン消費 0.5% 以下、残りを実作業に回せる
- 現在の状態 / 進行中プロジェクト / 直近完了だけが見える、迷いなし

## 移行コスト

- 初回フル実行: **約 4 時間**（Phase 1-6）
- 月次メンテ: **約 30 分**
- 四半期リストラクチャ: **約 2 時間**

## いつやるべきか

| 状態 | アクション |
|---|---|
| CLAUDE.md 500 行未満 | まだ必要なし、様子見 |
| 500-1000 行 | Phase 0 (分析) 実施、計画立てる |
| 1000-1500 行 | Phase 1-3 を数日かけて実行 |
| 1500+ 行 | **即座に** Phase 1-6 フルサイクル |
| 症状あり (AI が指示無視) | 行数関係なく即実行 |
