## 発見事項

### 工数設定は環境ごとに独立
- ローカルでMaxに設定しても、クラウド環境に切り替えるとリセットされる
- 環境を切り替えるたびに工数の再設定が必要

### クラウド環境のネットワーク制限
- サンドボックス化されたLinuxコンテナ
- SSHクライアントはインストール可能
- **Tailscale IP（100.x.x.x）に到達不可**（Tailnetの外）
- グローバルIPのサーバーには接続可能
- 実質GitHub操作 + Routines実行のみ

## 結論: 2層構成が最適

| 層 | 環境 | 用途 |
|---|------|------|
| 1 | Mac CLI | 全操作（ローカル+SSH+Tailscale+配信） |
| 2 | クラウド | GitHub操作 + Routines実行 |

### デスクトップアプリの評価（4/14判断を維持）
- Code モード統合、統計ダッシュボード、Effort UIなど改善あり
- しかしCLIで全て代替可能
- Electron製でM1 8GBには重すぎる（load 85の実績）
- Dispatchの差別化もSSH+CLIで代替済み
- **結論: 不要（リソースコストに見合わない）**
