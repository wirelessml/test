# Cowork用 SKILL.md 活用ガイド

## 概要

SKILL.mdを1枚入れるだけで、Coworkが毎回同じ品質で動く。
一度作れば、毎回の説明がゼロになり、呼びかけだけで同じ品質が毎回出る。

## SKILL.mdとは

| ファイル | 役割 |
|---------|------|
| CLAUDE.md | プロジェクト全体のコンテキスト |
| SKILL.md | 特定タスクの手順書・スキル定義 |

## 配置場所

```
.claude/skills/my-workflow.md
```

## 基本テンプレート

```markdown
---
name: my-workflow
description: "（このスキルが何をするか1行で書く。例: X投稿の下書きを3パターン生成する）"
---

# 役割
あなたは（職種・役割）の専門家です。
（ターゲット）に向けて（目的）を達成するために動きます。

## 禁止事項
- （やってはいけないこと1）
- （やってはいけないこと2）
- 抽象的な表現は使わない（具体的な固有名詞・数字を必ず入れる）

## 出力フォーマット
（出力の形式・文字数・構成を書く）

## 参照ファイル
- knowledge/（参考にするファイル名）
- memory/（記憶させたいファイル名）
```

## 使い方

1. 上のテンプレートをコピーして自分の情報に書き換える
2. `.claude/skills/` フォルダに保存
3. Coworkから呼びかけるだけで、定義した通りに動く

## Cowork vs Antigravity の SKILL.md の違い

中身の書き方は共通の思想で作れる。違いはフォルダの置き場所だけ。

| | Cowork | Antigravity |
|---|---|---|
| 配置場所 | `.claude/skills/スキル名/SKILL.md` | `.agent/skills/スキル名/SKILL.md` |
| 読み込み | Coworkが自動判断して読み込む | Antigravityのエージェントが参照 |
| 中身の書き方 | 自然言語のワークフロー定義 | ほぼ同じ構造 |

Cowork用に作ったSKILL.mdをAntigravityに転用するのに5分かからない。

## SKILL.mdで絶対やってはいけないこと3つ

### 1. description欄を空白にする

Coworkがスキルを自動選択できなくなる。「いつ使うか」を1行で具体的に書く。

### 2. 禁止事項を書かない

毎回同じ失敗を繰り返す。過去に失敗したパターンを禁止事項に書き続けると、スキルが自己改善していく。

### 3. 1つのSKILL.mdに全部詰め込む

「X投稿」「競合分析」「記事執筆」を1ファイルに混ぜない。タスクごとに1ファイルに分ける。Coworkは複数スキルを組み合わせて動ける。

## メリット

- 毎回の指示出しが不要になる
- 出力品質が安定する
- チームで共有すれば誰でも同じ品質で作業できる
