1. ボールト基礎

フォルダ構造（PARA + Zettelkasten）

Obsidianボールトの基本はフォルダ構造。PROJECTSに進行中の仕事、AREASに継続的な関心領域、RESOURCESに参照素材、ARCHIVEに完了物を置く。

Vault/
  00-Inbox/ ... 未整理のメモ
  10-Projects/ ... 進行中プロジェクト
  20-Areas/ ... 継続的な関心領域
  30-Resources/ ... 参照素材・ナレッジ
  40-Archive/ ... 完了・非アクティブ
  Templates/ ... テンプレート
  Attachments/ ... 画像・PDF等

2. Claude 統合

CLAUDE.md 設定

ボールトのルートに CLAUDE.md を置くことで、Claude Codeにプロジェクトコンテキストを渡す。ボールトの構造、命名規則、ワークフローを記述する。

# My Knowledge Vault

## ボールト構造
- 00-Inbox: 未処理メモ
- 10-Projects: アクティブプロジェクト
- 30-Resources: ナレッジベース

## 命名規則
- ファイル名: ケバブケース（例: claude-code-tips.md）
- タグ: スネークケース（例: #ai_tools）

## Claudeへの指示
- ノート作成時は必ずYAMLフロントマターを付与
- 既存ノートへのリンク [[]] を積極的に使う
- 新規ノートは00-Inboxに作成

Claude Code / デスクトップ セットアップ

  • 1
    ボールトをカレントディレクトリに
    cd ~/Documents/MyVault && claude で起動。CLAUDE.mdが自動読み込み。
  • 2
    ターミナルプラグイン連携
    Obsidian内蔵ターミナルからClaude Codeを直接起動。ノート操作がシームレス。
  • 3
    Claudeデスクトップアプリ
    MCPサーバー経由でファイルシステムアクセス。ボールト全体をコンテキストに。

MCP ツール & スキル

MCPサーバーでClaudeの能力を拡張。ファイル操作、Web検索、外部API連携が可能。

filesystem

ボールト内ファイルの読み書き・検索

memory

セッション跨ぎの記憶保持

web-search

リアルタイム情報の取得

google-photos

写真の検索・分類・整理

.claude/commands フォルダ

カスタムスラッシュコマンドを定義。繰り返しのワークフローを1コマンドで実行。

# .claude/commands/process-inbox.md
Inboxフォルダの未処理ノートを確認し、
以下の基準で分類してください:
- タグ付け（内容に基づく）
- 適切なフォルダへ移動
- 関連ノートへのリンク追加
- MOCへの追記

コンテキスト読み込み戦略

ボールト全体を毎回読み込むのは非効率。MOCをエントリポイントにして、必要なノートだけ段階的に読み込む「漏斗型」戦略が最適。
  • 1
    CLAUDE.md
    プロジェクト全体の構造とルール（自動読み込み）
  • 2
    MOC / ハブノート
    トピックの全体像と関連リンク一覧
  • 3
    個別ノート
    必要に応じてRead。過剰読み込みを避ける

3. コアプラグイン

必須プラグイン一覧

  • T
    Templater
    動的テンプレート。日付変数、条件分岐、JavaScriptスニペット対応。QuickAddと組み合わせて最強。
  • D
    Dataview
    SQLライクなクエリでノートを動的集計。タスク管理、進捗追跡、統計表示に。
  • Q
    QuickAdd
    マクロ・キャプチャ・テンプレート適用をホットキー1つで。Inboxへの高速メモ投入に最適。
  • P
    Periodic Notes
    日次・週次・月次・四半期・年次ノートを自動生成。レビューサイクルの土台。
  • C
    Canvas
    ビジュアルボード。ノート・画像・リンクを自由配置。ブレインストーミングやプロジェクト設計に。
  • G
    Graph View（コア機能）
    ノート間の関係を可視化。孤立ノートの発見、知識クラスターの把握に。

ノート処理プロンプト

1. YAMLフロントマターの補完（tags, type, status）
2. 見出し構造の整理
3. 関連する既存ノートへの [[リンク]] 追加
4. 要約セクションの追加
5. 未解決の疑問を ## Questions に記載

振り返り & 統合

# .claude/commands/weekly-review.md
今週のデイリーノートを読み、以下を生成:
1. 週のハイライト（3-5項目）
2. 学んだこと（知識の差分）
3. 未完了タスクの棚卸し
4. 来週の優先事項
5. ボールト内の関連ノートとの新しい接続

5. 知識ワークフロー

キャプチャ → 処理 → 接続

知識の流れを3段階で管理する。

  • 1
    キャプチャ（Inbox）
    思いついたこと、読んだもの、聞いたことを素早くメモ。形式は問わない。QuickAddで高速投入。
  • 2
    処理（Process）
    Inboxを定期的に処理。タグ付け、フロントマター追加、適切なフォルダへ移動。Claudeに一括処理させる。
  • 3
    接続（Connect）
    既存ノートとのリンク作成、MOC更新、新しいパターンの発見。ここが知識の価値が生まれる段階。

文献ノート vs 永続ノート

文献ノート

本・記事・動画の内容を自分の言葉で要約。原文の引用を最小限に。出典を必ず記録。

永続ノート

自分の思考・洞察を1ノート1概念で記述。文献ノートから抽出した「原則」や「パターン」。

Claudeに「この記事から永続ノートを3つ抽出して」と依頼すると、文献ノートから原則を自動抽出できる。

プロジェクト管理

Dataviewでプロジェクト一覧を動的生成。ステータス・期限・優先度でフィルタリング。

```dataview
TASK
FROM #project AND !"40-Archive"
WHERE !completed
GROUP BY file.link
```

6. 自動化

カスタム Claude スキル

.claude/commands/ にMarkdownファイルを置くだけでスラッシュコマンドとして使える。

  • /
    /tag-all
    フォルダ内の全ノートに内容ベースで自動タグ付け
  • /
    /link-orphans
    孤立ノートを検出し、関連ノートへのリンクを提案
  • /
    /summarize-folder
    フォルダ内の全ノートを要約してMOCを自動生成
  • /
    /daily-review
    今日のノートを振り返り、明日のタスクを整理

自動タグ付け & リンク

# .claude/commands/auto-link.md
30-Resources/ 内の全.mdファイルを走査し:
1. 他ノートのタイトルが本文に出現 → [[リンク]] に変換
2. 内容からタグを推定 → フロントマターに追加
3. 関連度の高いノートペアをリストアップ
4. MOCに未登録のノートを追記

デイリーレビュー自動化

cronまたはPeriodicNotesのフックで、毎日決まった時間にClaudeが前日のノートをレビュー。

# 毎朝9時に自動レビュー
0 9 * * * cd ~/Vault && claude "/daily-review" --print

8. 高度なテクニック

エージェント的ノート取り

Claudeをナレッジエージェントとして運用。ボールトを自律的に探索・整理・拡張させる。

# エージェント的ワークフロー
claude "ボールト内の #concept タグのノートを全て読み、
共通するパターンを3つ特定して、
それぞれ新しい永続ノートとして作成してください。
MOCも更新してください。"

多段階推論

複雑な問題を段階的に分解し、各段階の思考をノートとして保存。思考プロセス自体をナレッジにする。

  • 1
    問題定義ノート
    何を解決したいのか、制約条件は何か
  • 2
    仮説ノート群
    考えられるアプローチを個別ノートで展開
  • 3
    検証ノート
    各仮説の検証結果と判断根拠
  • 4
    結論ノート
    最終的な答えと、そこに至るリンクチェーン

ボールト・アズ・コンテキスト工学

ボールト全体をClaude向けのコンテキストとして設計する考え方。

ノートの粒度、命名、タグ体系のすべてが「Claudeが効率的に必要な情報にアクセスできるか」を基準に設計される。人間にもAIにも読みやすいノートが最強。

クロスノート分析

# 複数ノートの横断分析
claude "以下の5つのノートを読んで、
矛盾している記述があれば指摘し、
統合的な見解をまとめてください:
- [[メモA]]
- [[メモB]]
- [[メモC]]
- [[メモD]]
- [[メモE]]"

9. 可視化

グラフビュー最適化

  • -
    フィルタリング
    タグ・パスでフィルタして特定領域のグラフだけ表示。全体表示は大規模ボールトでは重い。
  • -
    グループカラー

  • -
    孤立ノート発見

10. メンテナンス & 最適化

ボールトヘルスチェック

# .claude/commands/vault-health.md
ボールトの健康状態を診断してください:
1. 孤立ノート数（リンクなし）
2. デッドリンク（存在しないノートへの [[]] ）
3. 空ノート
4. タグなしノート
5. 30日以上未更新のInboxノート
6. 重複タイトル

バックアップ & Git 同期

  • -
    Obsidian Git プラグイン
    自動コミット（10分間隔）。プッシュ/プルもスケジュール可能。
  • -
    .gitignore 設定
    .obsidian/workspace.json, .obsidian/cache, .trash/ を除外。プラグイン設定は含める。
  • -
    マルチデバイス同期
    Git + iCloud/Syncthing。コンフリクト時はGit優先。

Claude コンテキスト最適化

大きなボールト（1000+ノート）ではClaude Codeのコンテキストウィンドウに注意。MOC経由の段階的読み込み、Grep/Globでの絞り込みを活用する。全ノートを一度に読ませるのはNG。

パフォーマンスチューニング

  • -
    プラグイン最小化
    使わないプラグインは無効化。起動速度に直結。
  • -
    添付ファイル管理
    画像はAttachmentsフォルダに集約。巨大ファイルはGit LFSまたは外部リンク。
  • -
    Dataviewキャッシュ
    重いクエリはdataviewjsでキャッシュ。ページ読み込みが遅い場合に有効。
