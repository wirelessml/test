# AI時代の情報管理戦略：GitHubにすべてを集約する

> コンテキストを整備する者が、AIを最も活用できる。

---

## 原則：すべての情報をAIが参照できる場所にまとめる

### なぜGitHub一択なのか

| 比較項目 | GitHub | Notion | Google Docs | ローカルファイル |
|----------|--------|--------|-------------|----------------|
| AI対応 | 全エージェント対応 | MCP設定必要 | MCP設定必要 | そのマシンのみ |
| バージョン管理 | git標準 | 手動 | 手動 | なし |
| ツールロックイン | なし | あり | あり | なし |
| 検索性 | Grep/検索API | 独自検索 | 独自検索 | ローカル検索 |
| 共同編集 | PR/ブランチ | リアルタイム | リアルタイム | なし |

**結論**: Claude Code、Cursor、GitHub Copilot、Windsurf... どのAIエージェントに乗り換えても、GitHubなら即座に使える。

### まずは質ではなく量が大事

完璧なドキュメントを作ろうとして何も書かないより、雑でもいいからとにかく入れる。AIが整形・要約・検索してくれるので、人間が読みやすい形式である必要すらない。

---

## 実践：何をGitHubに置くか

### 1. プロジェクトコンテキスト（CLAUDE.md）

```markdown
# CLAUDE.md
- リポジトリの概要
- ディレクトリ構成
- コーディング規約
- よく使うコマンド
```

AIがセッション開始時に自動で読む。プロジェクト固有のルールをここに書く。

### 2. ナレッジベース（docs/）

```
docs/
├── セットアップガイド
├── トラブルシューティング
├── ツール比較・選定理由
├── 過去の意思決定ログ
└── 学習メモ・Tips
```

過去にハマったこと、解決策、なぜその技術を選んだか。未来の自分（とAI）への贈り物。

### 3. 会議情報（最も価値が高い）

```
meetings/
├── 2025-03-25-sprint-review.md
├── 2025-03-24-design-discussion.md
└── ...
```

**tl;dv** 等の文字起こしツールで全メンバーの発言を毎日GitHubに自動保存。

なぜ会議情報が最も価値が高いか：
- **暗黙知の宝庫** — ドキュメントに書かれない意思決定の背景がある
- **コンテキストが豊富** — 「なぜそうしたか」が議論の流れから分かる
- **全文検索可能** — AIが過去の議論を横断検索できる

### 4. テンプレート・ワークフロー（templates/）

```
templates/
├── KNOWLEDGE.md          ← ワークフローの手順書
├── 実際のテンプレートファイル
└── スクリプト
```

繰り返し行う作業のパターンを蓄積。AIはこれを参照して同じ品質で再現できる。

## 自動化パイプライン

### tl;dv → GitHub 自動保存

```
会議（Zoom/Google Meet）
    ↓
tl;dv が文字起こし
    ↓
Webhook / GitHub Actions
    ↓
meetings/ ディレクトリに自動コミット
    ↓
AIが過去の議論を参照可能に
```

### Claude Subconscious（セッション記憶の自動蓄積）

```
Claude Code セッション
    ↓
Subconscious が観察・学習
    ↓
好み・パターン・未完了タスクを記憶
    ↓
次のセッションで自動反映
```

### Insanely Fast Whisper（ローカル高速文字起こし）

```
音声ファイル（会議録音等）
    ↓
Insanely Fast Whisper でローカル文字起こし
    ↓
meetings/ ディレクトリに保存
    ↓
AIが過去の議論を参照可能に
```

- **速度**: 150分の音声を98秒で処理
- **モデル**: OpenAI Whisper Large v3
- **コスト**: 無料（ローカル実行、API不要）
- **対応HW**: Nvidia GPU / Apple Silicon (Mac)
- **機能**: 話者識別、タイムスタンプ、多言語対応

```bash
# インストール
pipx install insanely-fast-whisper

# 基本的な使い方
insanely-fast-whisper --file-name meeting.mp3

# 日本語指定 + 話者識別
insanely-fast-whisper --file-name meeting.mp3 --language ja --diarize
```

**GitHub**: github.com/Vaibhavs10/insanely-fast-whisper

#### 文字起こしツール比較

## アンチパターン

### やってはいけないこと

| アンチパターン | 問題 | 対策 |
|---------------|------|------|
| 情報をNotionだけに置く | AIエージェント切替時に使えない | GitHubにも同期 |
| 完璧なドキュメントを目指す | 結局何も書かれない | 雑でもいいから入れる |
| ローカルにメモを溜める | 他のマシン・AIから見えない | GitHubにpush |
| 会議メモを取らない | 暗黙知が消える | tl;dvで自動化 |
| CLAUDE.mdを書かない | AIが毎回ゼロから理解 | 最低限のコンテキストを書く |

---

## この戦略のROI

```
初期コスト：低い（GitHubリポジトリを作ってファイルを置くだけ）
        ↓
蓄積するほど：AIの出力品質が上がる
        ↓
最終的に：AIが過去の全コンテキストを持った「チームメンバー」になる
```

情報を溜める場所さえ正しければ、AIの進化に自動で乗れる。ツールが変わっても、情報は残る。

## 他社AI動向メモ

### Google Gemini — ケア記録アシスト Gem（2026年3月）

- **対象**: 介護従事者
- **機能**: Geminiアプリの「Gem」機能に「ケア記録アシスト」が追加
- **効果**: 介護記録の作成時間を約20%削減
- **特徴**: 複数の介護現場を訪問し、介護従事者のフィードバックを集めて設計
- **意義**: AIの業務特化型アプリケーションが介護分野にも拡大。特定業種向けのプリセットGem（カスタムエージェント）は、Claude CodeのSKILL.mdやCopilot Coworkのエージェントと同じ「業務特化AI」のトレンド

### Claude Code Usage Report プロンプト（2026年3月）

X（Twitter）で利用統計ダッシュボードを共有するためのプロンプトテンプレート。
`/stats` のスクリーンショットを添付し、以下の構成でHTMLダッシュボードを自動生成：

1. ヘッダー（Top X%バッジ + プランバッジ）
2. ティアバナー（ランク表示 + サマリー）
3. KPIカード4列（トークン数・API換算・日平均・節約額）
4. Tokens per Day棒グラフ（Chart.js、モデル別色分け + Phase分析）
5. 用途分析 + Key Insights（ベンチマーク比較リングチャート付き）
6. Model Usage / API換算コスト内訳 / トークン詳細テーブル / CTA

技術要件: Tailwind CSS + Chart.js + Inter + JetBrains Mono、単一HTMLファイル

### Mistral — オープンソース音声AI（2026年3月）

- **概要**: ElevenLabsを超えると話題のオープンソース音声生成モデル
- **最大の特徴**: モデル自体を公開（オープンソース）・API不要・何時間使っても0円
- **必要スペック**: RAM 3GB（ローカル動作）
- **機能**:
  - 感情付きの自然な音声生成
  - リアルタイム生成レベルの低遅延
  - 音声クローン対応
- **従来との違い**:

| 項目 | ElevenLabs等 | Mistral音声AI |
|------|-------------|--------------|
| 料金 | 従量課金 | 完全無料 |
| 動作環境 | クラウドAPI | ローカル |
| ソースコード | 非公開 | オープンソース |
| プライバシー | データ送信あり | ローカル完結 |

- **意義**: 音声生成の「民主化」。従量課金だった領域が無料・ローカル化。Claude CodeのInsanely Fast Whisper（文字起こし）と組み合わせれば、音声入力→処理→音声出力の完全ローカルパイプラインが構築可能

### Google Drive スキャン + NotebookLM 活用法（中学受験・教育）

- **課題**: 塾テキストをスマホで撮影→AIに読ませると文字が潰れて読み間違える
- **解決策**: Google Driveのスキャン機能を使う（無料）
  - スマホ撮影より圧倒的に高品質なスキャン
  - 文字認識エラーが9割解消
- **ワークフロー**:
  1. 寝る前に5分でテキストをGoogle Driveスキャン
  2. 翌朝NotebookLMに取り込み
  3. NotebookLMが子供に解説（音声解説も可能）
- **ポイント**: S塾（SAPIX）等のテキストは複雑なレイアウトが多いため、スキャン品質が重要
- **応用**: 学校プリント・塾教材のHTML化（当リポジトリの `templates/KNOWLEDGE.md` も参照）
