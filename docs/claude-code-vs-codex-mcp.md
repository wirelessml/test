# Claude Code vs Codex（OpenAI）MCP比較 — 実務者目線

## 記録日

2026年3月30日

## 結論

MCP対応の有無だけでなく「実務で使えるMCPサーバーがどれだけ揃っているか」と「非エンジニアでも使えるか」の差が大きい。現時点ではClaude Codeが選ばれやすい。

## Claude Codeが選ばれる3つの理由

### 1. MCPサーバーの充実度

税理士の実務に関わるサービスが既にClaude Code向けにMCPサーバーを公開済み：

- **freee** — 公式MCPサーバー提供済み。すぐに使える
- **マネーフォワード** — MCP対応
- **Notion** — MCP対応
- **Gmail** — MCP対応
- **Googleカレンダー** — MCP対応

Codex側はまだこのエコシステムが追いついていない。

### 2. 設定の簡単さ

- 設定ファイル（claude_desktop_config.json）にMCPサーバーのURLを書くだけで接続可能
- 非エンジニアの税理士でも数分で「freeeの試算表を取得して」が動く状態になる

### 3. 対話型エージェントとしての設計思想

- ターミナル上で「会話しながら業務を進める」設計
- ツール横断の連続操作が自然にできる：
  - freeeで売上確認 → Gmailで報告メール作成 → カレンダーに次回MTG登録
- Codexはリポジトリ内のコード生成・編集が主戦場で、この"ツール横断"の体験が異なる

## トークン節約術（AI駆動開発の一部）

Proプランでの運用で学んだ節約術：

1. **軽いタスクはSonnetで** — `/model claude-sonnet-4-6` に切り替え
2. **画像の送信を減らす** — スクリーンショットはトークン消費が大きい
3. **質問はまとめて送る** — 1往復で複数の質問をする
4. **SKILL.mdを活用** — 毎回の指示を減らせば入力トークンが減る
5. **CLAUDE.mdにコンテキストを蓄積** — 繰り返しの説明が不要になる
