# Claude Computer Use セットアップ & 利用ガイド

> 2025年3月23日発表。Claude がユーザーの許可のもと Mac を直接操作し、デスクワークを代行する機能。

> **⚠ 現在のステータス：リサーチプレビュー**
> - 対象プラン：Pro / Max のみ
> - 対応OS：**macOS Sonoma（14.0）以降**（Computer Use の画面操作）
> - **Windows**：Coworkは対応済み（2026年2月〜）、Computer Use（画面操作）は「coming soon」
> - Claude デスクトップアプリの起動が必須
> - 正式リリースではないため、機能や仕様が変更される可能性がある

> **🔥 2026年3月30日 速報：Claude Code に Computer Use 搭載**
>
> Claude Code（CLI）から直接PC操作が可能に。

> **実際に動作した環境からのレビュー：**
>
> 操作の仕組み:
> - スクショベースでClaudeが画面を認識
> - 対象の座標認識後のクリック操作は爆速
> - MCPなのでClaude Code自身がツールを認識している
> - ドラッグ＆ドロップやバッチ操作も対応
>
> 承認・ウィンドウ管理:
> - アプリ単位で承認が必要（初回のみ、同セッション内はスキップ）
> - 操作対象以外のウィンドウは一時的に最小化される
> - アクティブウィンドウを持っていかれるので手を止めるのが無難
>
> 課題:
> - 電卓の数字ボタン位置を間違えるなど、座標認識の精度に課題
> - 操作ミス時にClaudeが認識・リカバリできる仕組み作りが重要
>
> 活用のポイント:
> - **定常的な操作、特に時間がかかるE2Eテストに任せる価値あり**
> - 操作ミスしてもClaudeが認識できる仕組み（エラーハンドリング）が大事
>
> **できること：**
> - CLIからアプリ起動・UI操作・テスト検証まで自動化
> - Swiftアプリをビルド→起動→全ボタンクリック→スクショまで1会話で完結
> - iOSシミュレータ操作、レイアウトバグ再現、GUI専用ツールの操作
> - Dispatch連携でスマホから指示も可能
> - Playwright設定もテストコードも不要
>
> **対応環境：** macOS、Pro/Max プラン（v2.1.85以降）、Team/Enterpriseは非対応
>
> **有効化手順：**
> 1. Claude Code内で `/mcp` を実行
> 2. 「computer-use」サーバーを選択してEnable
> 3. 初回使用時にmacOSの「アクセシビリティ」と「画面収録」の権限を許可
>
> **制限事項：** `-p` フラグの非インタラクティブモードでは使用不可
>
> **安全設計：**
> - 操作中は他のアプリが自動で非表示（終了後に復元）
> - ターミナルは常に表示されるがスクショ対象外（Claude自身の出力は読めない設計）
> - アプリごとにセッション単位で許可が必要
> - ターミナルやFinderなど高権限アプリは追加警告が出る
> - **Escキーでいつでも即中断可能**
> - マシン全体でロック → 同時に1セッションのみ
> - 画面上のプロンプトインジェクション検知を内蔵
> - センシティブなアプリは閉じてから使うことを推奨
>
> **アプリ種別ごとの操作権限：**
> | アプリ種別 | 操作レベル |
> |-----------|----------|
> | ブラウザ / 証券アプリ | 閲覧のみ |
> | ターミナル / IDE | クリックのみ |
> | その他 | フル操作 |
>
> **Claudeの操作優先順位（Computer Useは最後の手段）：**
> 1. MCP連携がある → MCPを使う
> 2. シェルコマンドで済む → Bashを使う
> 3. ブラウザ作業でChrome連携あり → Chrome経由
> 4. どれも使えない → **Computer Use**
>
> 画面操作は最も汎用的だが最も遅い。直接統合がないアプリ・シミュレータ・API非対応ツール向け。
>
> **意義：** 従来はデスクトップアプリのCowork経由でのみ可能だったPC操作が、
> CLIから直接可能に。理論上、大半の業務を自動化できるレベルに到達。

## 1. 前提条件

| 項目 | 要件 |
|------|------|
| **ステータス** | リサーチプレビュー（正式版ではない） |
| **プラン** | Claude Pro または Max |
| **OS** | **macOS Sonoma（14.0）以降**のみ（Windows / Linux / iOS 単体は未対応） |
| **アプリ** | Claude デスクトップアプリ（最新版）の起動が必須 |
| **ブラウザ操作** | Google Chrome（拡張機能が必要） |

---

## 2. 機能の概要

Claude の「**Cowork**」タブ内にある **Dispatch（ディスパッチ）** 機能を使うと、Claude が以下を実行できる：

- アプリの起動・切り替え
- ブラウザでの検索・ナビゲーション・フォーム入力
- スプレッドシートへの値入力
- ファイル操作（読み取り・書き込み）
- その他一般的なデスクトップ操作

### 重要な制約事項

| 制約 | 詳細 |
|------|------|
| **Computer Use は最終手段** | タスクを Computer Use なしで完了できる場合、Cowork は Computer Use を使わない。時間もリソースも無駄になるため、API呼び出しやコマンド実行等のより効率的な手段が優先される |
| **ターミナル入力は禁止** | Computer Use からターミナルに文字を入力することは許可されていない。ターミナル操作が必要な場合は Claude Code（CLI）側で行う |
| **Chrome 操作には拡張機能が必須** | Computer Use から Chrome に特定のページを表示させるには **Claude in Chrome 拡張機能**が必要。拡張機能なしではブラウザ操作ができない |

> **つまり**：Computer Use は「他に手段がないときの最終手段」であり、何でも画面操作で解決しようとするツールではない。Cowork は最も効率的な方法を自動で選択する。

## 3. セットアップ手順

### 3.0 Cowork デーモンのインストール（ターミナルからセットアップする場合）

GUI の Dispatch 設定画面を使わず、ターミナルから直接セットアップすることも可能。

```bash
# Cowork デーモンをインストール
curl -fsSL https://cowork.anthropic.com/install.sh | bash

# 起動して Claude app とペアリング
cowork start
# → ターミナルに QR コードが表示される
# スマホの Claude app でスキャンしてペアリング完了

# ステータス確認
cowork status
# → "Paired: YES  |  Status: Ready" が出れば設定完了
```

**ポイント**：
- スマホしか手元にない場合でも、SSH 等でリモートの Mac にアクセスして `cowork start` → スマホの Claude app で QR スキャンという流れでペアリングできる
- ペアリング後は Claude app 側から Dispatch タスクを投げられる

### 3.1 Claude デスクトップアプリを開く（GUI からセットアップする場合）

上部タブで **「Cowork」** を選択し、左サイドバーから **「Dispatch」** をクリック。

「ディスパッチの準備」画面が表示される。

### 3.2 各設定項目を有効化する

以下の順番で設定を進める：

#### ① Claudeにファイルへのアクセスを許可する

- トグルを **オン** にする
- 「完了」をクリックすると macOS がアクセシビリティの許可を求める
- **システム設定 > プライバシーとセキュリティ > ファイルとフォルダ** で Claude を許可

#### ② このコンピュータをスリープさせない

- Dispatch の実行中にスリープを防止する設定
- 長時間タスク（リサーチ、データ入力など）を任せる場合は **オン** を推奨

#### ③ Claude in Chrome をインストール

- 「インストール」ボタンをクリック
- Chrome 拡張機能がインストールされる
- これにより Claude がブラウザ内でナビゲート、クリック、フォーム入力を行えるようになる
- **注意**: Chrome が既定ブラウザとして設定されている必要がある

#### ④ Claudeにコンピュータの制御を許可

- トグルを **オン** にする
- macOS のアクセシビリティ許可が求められる
- **システム設定 > プライバシーとセキュリティ > アクセシビリティ** で Claude を許可
- これにより、クリック・キーボード入力・アプリの起動が可能になる

#### ⑤ コネクタの確認

- 「すべてのコネクタがオンです」と表示されていれば OK
- 認証済みの外部サービス（Google、Slack など）との連携状態を確認

### 3.3 設定完了の確認

すべての項目が有効になったら、左サイドバーの **「+ 新しいタスク」** からタスクを作成できる。

## 4. 基本的な使い方

### 4.1 タスクの作成

1. **「+ 新しいタスク」** をクリック
2. 自然言語でやりたいことを入力する（例：「Googleスプレッドシートを開いてA1にタイトルを入力して」）
3. Claude がタスクを分解し、ステップごとに実行する

### 4.2 タスクの例

| カテゴリ | 具体例 |
|---------|--------|
| **ブラウザ操作** | 「〇〇のサイトを開いて情報をまとめて」 |
| **データ入力** | 「このCSVの内容をGoogle Sheetsに転記して」 |
| **リサーチ** | 「競合3社の料金プランを調べて比較表を作って」 |
| **ファイル管理** | 「デスクトップのPDFを読んで要約して」 |
| **アプリ操作** | 「Numbersを開いて新しいシートを作成して」 |

### 4.3 タスクのスケジュール

- 左サイドバーの **「予定済み」** からタスクの予約実行が可能
- 定期的なタスク（日次レポート作成など）に活用できる

## 5. macOS の権限設定まとめ

Computer Use を利用するには、macOS のシステム設定で以下の権限を付与する必要がある：

```
システム設定 > プライバシーとセキュリティ
```

| 権限カテゴリ | 用途 |
|-------------|------|
| **アクセシビリティ** | クリック・キーボード入力・アプリ制御 |
| **ファイルとフォルダ** | ファイルの読み書き |
| **画面収録**（必要に応じて） | 画面の状態を認識するため |

---

## 6. セキュリティ上の注意点

- Claude は **ユーザーの許可なく** 操作を行わない。各操作前に確認が入る
- パスワード入力や決済操作は慎重に扱うこと
- 機密情報が含まれるアプリ・サイトを操作させる場合はタスク内容を限定する
- 「コンピュータの制御を許可」は使わないときは **オフ** にしておくことを推奨
- Chrome 拡張経由の操作はブラウザのシークレットモードでは動作しない

## 7. トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| **「コンピューター使用」トグルが表示されない** | アプリの言語設定を**英語に変更** → ログアウト → アプリを終了 → 再起動して再ログイン（下記参照） |
| タスクが実行されない | macOS のアクセシビリティ権限を再確認 |
| ブラウザ操作ができない | Chrome 拡張「Claude in Chrome」がインストール済みか確認 |
| 途中でスリープする | 「このコンピュータをスリープさせない」をオンにする |
| 画面認識がおかしい | ディスプレイ解像度の変更やマルチモニター構成を見直す |
| コネクタが認証切れ | Dispatch 設定画面でコネクタを再認証 |

### 「コンピューター使用」トグルが表示されない場合

日本語など英語以外の言語設定では、翻訳データが未提供のため Computer Use のトグルが非表示になることがある。

**解決手順：**

1. Claude Desktop の **設定 → 言語** を **英語（English）** に変更
2. **ログアウト** する
3. アプリを **完全に終了** する（Dock からも閉じる）
4. アプリを **再起動** して **再ログイン**
5. Settings → 「Computer use」トグルが表示される

> **補足**: 言語を英語にした後、UIが英語表示になるがComputer Useの設定完了後に日本語に戻してもトグルは維持される場合がある（要検証）。

## 8. Cowork タブの構成

Claude デスクトップアプリの上部タブは3つ：

| タブ | 機能 |
|------|------|
| **チャット** | 従来のテキストチャット |
| **Cowork** | Computer Use（Dispatch）によるデスクトップ操作 |
| **コード** | Claude Code（コーディング支援） |
