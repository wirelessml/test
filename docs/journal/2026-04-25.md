### 13:22〜13:36 CrystalDiskInfo に SanDisk が見えない問題発覚

- Windows 起動成功後、ヘルス確認のため CrystalDiskInfo 9.8.0 を起動
- **Seagate ST2000LM015（D: ドライブ）は表示**されるが **SanDisk Extreme V2（C: ドライブ）が表示されない**
- 原因: 内部の **ASMedia ASM2362 USB-NVMe ブリッジ**が SMART コマンドを汎用ツール経由で渡してこない

## 重要な学び（ナレッジ保存）

### 1. ASMedia ASM2362 USB-NVMe ブリッジの SMART 取得は専用ツール必須

- **CrystalDiskInfo 9.8.0（最新）でも認識しない**（USB/IEEE 1394 + 全翻訳ドライバ有効でも無理）
- **smartmontools `-d sntasmedia` も成功率低い**（試してないが既知の限界）
- **SanDisk Dashboard が唯一の正解**（独自プロトコルで ASM2362 と通信）
- ただし Dashboard でも **温度・インターフェース速度は取得不可**（ブリッジ FW の物理的限界）

### 2. SanDisk Dashboard 5.2.2.3（2026-03-05 リリース）の特性

- 公式注記「外付けは一部非対応」だが **Extreme V2 は対応していた**
- Online Installer (3.1 MB) / Offline Installer (111 MB)
- Windows 10/11 32/64-bit
- SMART 簡易診断（5 分）/ 拡張診断（10+ 分）機能あり
- TRIM / 書き込みキャッシュ管理 / FW 更新 / Windows ツールへのショートカット

### 3. GIGABYTE Z370N WIFI で USB-C SSD ブートする条件

- **CSM サポート 無効**（UEFI のみ）
- **XHCI Hand-off 有効**（最重要、これが root cause だった）
- Secure Boot は適切な署名 boot loader 必要
- **6 時間半の試行錯誤**を経た正解構成

### 4. Windows USB ドライバ Boot Critical 化の手順

Phase 1-2.5 で確立:
- USB Services の `Start=0` 設定
- USB Driver の Boot Critical 化
- `PortableOperatingSystem` フラグセット
- `BCD Rebuild` 実行
- `MountedDevices` レジストリリセット

### 5. CrystalDiskInfo USB バス検出の落とし穴

「機能」→「上級者向け機能」配下に **3 段階の有効化が必要**:
1. アドバンスドディスクサーチ
2. **USB/IEEE 1394** ← ここが OFF だと USB ドライブ全般がスキャンされない
3. USB/IEEE 1394 配下の各**翻訳ドライバ**（ASMedia/Phison 含む）

ただし 3 段階全て ON にしても、ブリッジ FW が独自プロトコルなら結局取れない（今回の ASM2362 ケース）。

### 16:00〜17:00 維新の嵐 起動格闘 → AcGenral.DLL 真犯人発見

- Windows 11 native + 互換モードで SETUP.EXE 起動成功（タイトル画面クリア）
- インストール完了後、ゲーム起動時に「**CD-ROM が見つかりません**」エラー多発
- 当初仮説（ドライブレター不一致 F:↔G:、設定 INI 探索）すべて外れ
- **真犯人**: **Win95 互換モードが Win11 の AcGenral.DLL（AppCompat シム）でクラッシュ**
  - 検出方法: `C:\ProgramData\Microsoft\Windows\WER\ReportArchive\AppCrash_ISHIN2.EXE_*` の `Report.wer` 開いて `Sig[3] = 障害モジュール=AcGenral.DLL` 一発で判明
  - **Procmon 不要、WER で十分**だった
- 互換モード `Win95` → `WINXPSP3` 変更で解決
- 全 EXE (ISHIN2.EXE / ISHIN2CF.EXE / AUTORUN.EXE) に統一互換層適用:
  ```
  ~ DWM8And16BitMitigation DISABLEDXMAXIMIZEDWINDOWEDMODE RUNASADMIN 256COLOR WINXPSP3
  ```
- **タイトル画面 + BGM (CD-DA Track 2/3) 完全動作**

### 維新の嵐 関連の重要ナレッジ

- CD はボリュームラベル `ISHIN2` で自動検出 → ドライブレター無依存
- BIN/CUE の構造（data + audio2 + audio3）が必須、ISO だと BGM 落ちる
- 設定保存先: `HKCU\Software\KOEI\ISHIN2\1.0`（CD パスは保存しない、起動毎全ドライブ走査）
- DirectX 5 redist インストールは **絶対禁忌**（Win11 内蔵 DX12 で十分）
- `RUNASADMIN` 必須（UAC VirtualStore 経由を回避）
- タイトル画面放置でも 1 コア常時 100%（1998 年代お約束の入力 spin-loop）→ ノート PC ではバッテリー禿げ案件

## 重要な学び（Phase 2-3 追加分）

### 6. 1998 年 KOEI ゲーム動作 7 訓
1. 「CD-ROM が見つかりません」表示 ≠ CD 未検出（互換シムクラッシュの煙幕の可能性）
2. WER `Report.wer` の `Sig[3]` で障害モジュール一発特定（procmon 不要）
3. Win95/98/Me 互換モードは Win10/11 で AcGenral.DLL を高確率で殺す → **WINXPSP3 が安定**
4. 関連 EXE 全てに**同一互換層文字列**を適用（差分があるとシム不整合）
5. 1998 年 KOEI ゲームはボリュームラベルで CD 自動検出（ドライブレター無依存）→ 仮想マウントで OK
6. CD 音楽トラック付きゲームは BIN/CUE 必須、ISO は不可
7. DirectX 5 redist は禁忌（Win11 DX12 で十分）

### 7. Claude Code on Windows の 2 つの罠
1. **PowerShell 実行ポリシー** → `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned -Force`
2. **git-bash 必須** → `winget install --id Git.Git`
（詳細手順書: @docs/guides/claude-code-windows-install.md）

### 8. AI 2 体協働の運用パターン
- 共有チャネル（HTTP API + JSON 永続化）+ 各機で `/loop` cron による自律ポーリング
- 役割分担: 現地実行担当 ↔ 提案統括担当
- 人間ユーザーは司令官役（要所介入のみ、無関与時間が長い）
- Mac/Win 同じ Anthropic アカウント、使用枠は共有
- AI のミス（自分の bg タスクが原因）も率直に自白するのでデバッグ早い

### 9. しゅん先生 PC 騒音原因 3 大要因
1. **HDD スピン**: `powercfg` で `disk-timeout` 短縮で解決
2. **Defender スケジュールスキャン**: 大容量バックアップ `.adi` を毎回解凍 → 例外追加で解決
3. **古いゲーム/アプリの spin-loop**: `Get-Process | Sort CPU -Desc` で発見 → Stop で解決
