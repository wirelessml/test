## 重要な学び（ナレッジ保存）

### 1. Yahoo Japan の UI/URL 構造刷新中（2024-2026）

- アカウント管理関連の URL が**ほぼ全部 404**
- 公開ヘルプドキュメントも CSS-error で読めない
- アクセス手段: メール本体（mail.yahoo.co.jp）から各機能を辿るしかない
- 教訓: **公式 URL を信じず、Yahoo メール内ナビゲーションを優先**

### 2. Yahoo メール 転送 vs IMAP/POP/SMTP は排他

- 転送 ON → IMAP/POP/SMTP 自動 OFF
- Yahoo の独自仕様（一般 IMAP プロバイダにはない）
- 教訓: Yahoo メールに対しては選択を強制される、両立不可

### 3. 転送設定の隠れた副作用

- 転送されたメールは **Yahoo サーバーから消える**（明記）
- iPhone Yahoo メールアプリで新着が見えなくなる
- 既存メールは残るが、新着は別環境（Gmail）で見るしかない

### 4. パスワード露出の即時対応プロトコル

仲さんが今日経験した重要パターン:
1. AI チャットにパスワードを入力してしまう
2. AI が即座に警告 + パスワード変更を強く推奨
3. ユーザーが即座にパスキー化（パスワード自体を不要に）
4. 結果: 露出済みパスワードは無効化、新パスワードはパスキーで保護

→ **「AI に頼んだらパスワードが安全な手順で扱われる」**ベストプラクティス事例として記録価値あり。

### 5. AI 経由のパスワード入力は仕組み的に拒否される

私が安全規則として:
- ❌ 「Never authorize password-based access to an account on the user's behalf」
- ❌ 「Never enter sensitive financial or identity information」
- ✅ ユーザーへの直接入力誘導のみ

教訓: パスワード入力を AI に頼むことはできない、仕様レベルの制約。

## 12:00 JST — Claude Code 月次制限疑惑（要追跡）

### 事象

iPhone のチャット画面上で **「You've hit your org's monthly usage limit」** エラーに遭遇。スクショあり。画面ヘッダーは「Mac test」、下部 UI は「コード」「マイク」「+」ボタンの構成（Maestri or Claude 系 iOS アプリ）。画面に勝間和代ツイート 2 本のリンクが映っていた:
- https://x.com/kazuyo_k/status/2048944369282191711
- https://x.com/kazuyo_k/status/2048739864594751575

### 会話の流れ

1. ユーザー: 「月間制限？」
2. Claude（誤回答）: 「Claude Code CLI に月間制限はない、5h + 週次のみ」（12:15 スナップショット: 週次 45%, 5h 1%）
3. ユーザーがスクショ提示（"You've hit your org's monthly usage limit"）
4. Claude（訂正）: 「Claude.ai consumer 側には月次クォータあり、文言は org でも個人 OK」
5. ユーザー: 「これはバグ？」
6. Claude: 「バグではない、Anthropic は個人も org 扱い」
7. **ユーザー: 「私は Claude Code CLI しか使ってない」** ← 矛盾発生
8. Claude: 「それなら本当におかしい、CLI に月次はないはず、スクショのアプリは何？」

### 矛盾点

| 観測 | 想定との差 |
|---|---|
| ユーザー主張: Claude Code CLI のみ使用 | 公式仕様では CLI に月次制限なし（5h + 週次のみ） |
| iPhone で月次制限エラー出現 | consumer Claude.ai 系の文言 |
| 「your org's」表記 | 個人アカウントでも org 文字列が出るのは普通 |
| 「Mac test」ヘッダー + コードボタン | Maestri らしい UI（Mac の Code CLI を iPhone から呼ぶリレー） |

### 仮説（優先度順）

1. **Maestri が claude.ai consumer API 経由でリレーしている** — Mac 側 Code CLI ではなく Maestri アプリ自体の API キー or claude.ai セッションを使ってる可能性。だとすれば月次制限は Maestri 側のサブスクで発生
2. **Anthropic が Code CLI にも月次制限を新規追加** — 2025 年後半〜2026 年初頭に静かに導入された可能性
3. **Code Mobile（公式 iOS アプリ）の独自制限** — CLI とは別計算
4. **表示バグ**: 週次制限が誤って monthly と表示されている
