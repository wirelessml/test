## 重要な学び（ナレッジ保存）

### 1. Yahoo Japan の UI/URL 構造刷新中（2024-2026）

- アカウント管理関連の URL が**ほぼ全部 404**
- 公開ヘルプドキュメントも CSS-error で読めない
- アクセス手段: メール本体（mail.yahoo.co.jp）から各機能を辿るしかない
- 教訓: **公式 URL を信じず、Yahoo メール内ナビゲーションを優先**

### 2. Yahoo メール 転送 vs IMAP/POP/SMTP は排他

- 転送 ON → IMAP/POP/SMTP 自動 OFF
- Yahoo の独自仕様（一般 IMAP プロバイダにはない）
- 教訓: Yahoo メールに対しては選択を強制される、両立不可

### 3. 転送設定の隠れた副作用

- 転送されたメールは **Yahoo サーバーから消える**（明記）
- iPhone Yahoo メールアプリで新着が見えなくなる
- 既存メールは残るが、新着は別環境（Gmail）で見るしかない

### 4. パスワード露出の即時対応プロトコル

仲さんが今日経験した重要パターン:
1. AI チャットにパスワードを入力してしまう

4. 結果: 露出済みパスワードは無効化、新パスワードはパスキーで保護

→ **「AI に頼んだらパスワードが安全な手順で扱われる」**ベストプラクティス事例として記録価値あり。

### 5. AI 経由のパスワード入力は仕組み的に拒否される

### 仮説（優先度順）

1. **Maestri が claude.ai consumer API 経由でリレーしている** — Mac 側 Code CLI ではなく Maestri アプリ自体の API キー or claude.ai セッションを使ってる可能性。だとすれば月次制限は Maestri 側のサブスクで発生

3. **Code Mobile（公式 iOS アプリ）の独自制限** — CLI とは別計算
4. **表示バグ**: 週次制限が誤って monthly と表示されている

### 12:30 解決 — 真因判明（仮説 1 の変種で確定）

ユーザー追加情報:「これは iPhone の Claude アプリ。リモートコントロールで claude code CLI に入ってます」

**確定構造（二重クォータ消費）**:

```
iPhone (Claude アプリ ≒ claude.ai consumer)
    ↓  ← claude.ai 月次クォータを消費（リモコン中継のため）
Anthropic クラウド（リモートコントロール経由）
    ↓  ← Mac 側 Code CLI の週次/5h クォータを消費
Mac (Claude Code CLI 本体)
```

**結論**: バグではなく**仕様**。Claude iOS アプリのリモートコントロール機能は **claude.ai consumer プランの月次クォータ**を消費する。たとえ実コンピュテーションが Mac 側 Code CLI で走っていても、メッセージ往復のリレーで claude.ai 月次が削られる。

- Mac 側 `/usage`（12:29 SSH スクショ）: 健全（5h 1%、週次 45%）
- iPhone 側（11:59 スクショ）: claude.ai consumer 月次が枯渇 → リモコン継続不可

### 回避策（優先度順）

1. **iPhone から純粋 SSH 直結**: Termius / Blink / a-Shell で Mac に SSH → Code CLI 直接操作（claude.ai 経由しない、月次消費ゼロ）。Tailscale + SSH 鍵は既に構築済
2. **Mac 前にいる時はリモコンを使わない** — Terminal 直接
3. **claude.ai 月次リセット待ち**（Settings > Usage で next reset 確認可）
4. **重い相談は Mac で完結**してから iPhone で結果確認のみ
