### 学んだこと

- **拗音は標準と外来語形の二系統**: `ja/ju/jo`（じゃじゅじょ）vs `je`（じぇ）。前者は IME wapuro 常識、後者はガジェット用語で頻出

- **Self-cap は streak 300 + done 30 で十分**: 300 連で +50% bonus は最大、それ以上やっても rank 据え置きで時間ロスのみ
- **2 つの cap criteria を OR で組む**: streak がまだ伸びてる時は streak 達成を待つ、伸び悩む時 (timeout) は諦めて次へ。両条件を同時 polling で扱うと 90 秒中の早期離脱と保険の両立可能
- **agent-browser `--headed` フラグでユーザーが視覚的に確認可能**: 自動化を「見える化」することでデバッグも UX もぐっと改善（ユーザーから「終わる気配がない」フィードバックがすぐ来た）
- **streak ボーナスがセッション時間を大幅延長**: 中級 round 2 で残時間 7:32 まで膨れ上がった（初期 2:00 から +5:30 以上）。自動 cap がないと検証時間が指数的に伸びる罠

### agent-browser 復活ループの罠

**現象**:
1. `agent-browser close --all` → CDP セッション切断、応答 "✓ Closed session: default"
2. `pkill -9 -f "agent-browser-chrome"` → chrome 群を SIGKILL
3. **数秒後に新 chrome 群が自動再起動**（PID 帯が新しい、user-data-dir UUID も別物）
4. ループ繰り返し → メモリ解放されず

**真犯人の特定**:
```bash
ps -eo pid,ppid,etime,rss,comm | grep agent-browser-darwin
# → 44314  1  00:38  10192  agent-browser-darwin-arm64
```
parent PID 1 の **孤立デーモン** `agent-browser-darwin-arm64` が裏で生き続け、chrome を自動 spawn し直していた。`close --all` は CDP 接続を閉じるだけで supervisor 自体は止めない仕様。

**完全停止コマンド**:
```bash
pkill -9 -f "agent-browser-darwin"   # supervisor を先に殺す
pkill -9 -f "Chrome for Testing"     # chrome 子プロセス群
pkill -9 -f "agent-browser-chrome"   # 念押し
```
3 コマンド連射で復活ループ停止。

### 学んだこと

- **CLI ツールが裏でデーモン化することがある**: `agent-browser` のように単発 CLI と思い込んでいたツールが parent PID 1 で daemon 化する設計は珍しくない（playwright server, headless chrome supervisor 等）。`ps -eo pid,ppid,etime` で **parent 1 + 長い etime** のプロセスを定期的に洗うのが衛生的
