### 10:40 SSD より速い技術の調査

ストレージ階層を整理:
1. CPU キャッシュ（<1ns）
2. HBM (3-5 TB/s) ← NVIDIA H100/B200 内蔵
3. DDR5 RAM（50-100 GB/s）
4. Optane / 3D XPoint（7 GB/s、10μs）← 2022 年 Intel 撤退、終売
5. PCIe Gen 5 NVMe（14 GB/s）← しゅん先生 PC（Gen3）では Gen3 速度に落ちる
6. PCIe Gen 3 NVMe（3.5 GB/s）← Plextor クラス
7. SATA SSD（0.56 GB/s）
8. HDD（0.1-0.25 GB/s）← 現 Seagate SMR

**しゅん先生 PC（Gen3）で実現可能な「SSD 以上」**:
- **RAM ディスク（tmpfs / ImDisk）**: 16GB のうち 4GB を R: ドライブ化 → **40 GB/s（NVMe の 10 倍速）**、揮発性、ブラウザキャッシュ置きに最適
- Optane 中古探索（入手困難、¥10,000-30,000）

## 発生した問題と対応

### 問題 1: Amazon API WebFetch が HTTP 500

**状況**: CronCreate 経由の Kioxia 監視で `WebFetch` が HTTP 500 エラー
**対応**: curl 直接取得にフォールバック（UA: Safari macOS 偽装、1.4MB 取得成功）
**学び**: Amazon は WebFetch の bot 検知が厳しい可能性、curl + proper UA が確実

### 問題 2: PowerShell SSH での日本語文字化け

**状況**: MASU-P55 SSH で `powershell -Command` 内の日本語クエリが cmd.exe に誤って渡される
**対応**: `iconv -t UTF-16LE | base64` で PowerShell `-EncodedCommand` フォーマット使用
**学び**: SSH 経由の Windows PowerShell は EncodedCommand が確実、ただし出力は CLIXML で Shift-JIS 混入のことあり

### 問題 3: Intel 80GB SSD の出所不明

**状況**: ユーザー主張「情報は Windows マシンにある」→ MASU-P55 に痕跡ゼロ
**原因**: ユーザー前自作 PC は 3.5" HDD のみ、この 2.5" SSD は別経路で入手
**対応**: しゅん先生 PC 側に情報ある可能性、Tailscale 導入後に再調査
