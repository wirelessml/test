## K. 学んだこと (本セッション)

1. **🚨 Codex のサンドボックス制限**: `~/.codex/` 配下への書き込みが `Operation not permitted` で阻止される。 staged は `~/Desktop/.tmp/hatch-pet-<id>/package/<id>/` に残るので、 ユーザー側で `cp` 代行配置すれば良い。
2. **🚨 spritesheet RGBA 必須**: RGB で保存すると Codex Desktop App はペット一覧から除外する。 Pillow 保存時は `RGBA` mode 維持 + `lossless=True` 推奨。
3. **Codex の image_gen は実用品質**: 抽象 chibi キャラを 9 ポーズ × 8 frame で生成 + 透過化 + 72 frame atlas 化までを 1 タスクで完遂 (149K tokens で約 5 分)。 Pillow 自作の stick figure とは比較にならない品質。
4. **PR merge 監視 cron**: GitHub PR の merge 状態は `gh pr view --json state,mergedAt` で取得可能。 LaunchAgent + macOS notification で自動化できる。
5. **codex update は brew cask 経由**: stable channel のみ追従、 alpha は GitHub release から手動 install が必要。
6. **codex MCP server 化**: `codex mcp-server` で Codex CLI 自体を MCP server として動かす機能あり。 Claude Code から `Called codex` ネイティブ表示で呼び出し可能。
7. **VNC 真っ黒問題 (macOS 26)**: `screensharingd` が TCC Screen Recording 権限を持っていないと、 VNC 接続成功しても画面 capture 失敗で真っ黒。 別途 sudo + 設定 UI で許可必要 (今回は保留)。
8. **「家」 vs 「氏」 の呼称感覚**: ミニマリスト思想で「家」 (持ち物)を否定、 「氏」 (血縁・系譜) を肯定する仲氏の語感。 「源氏」 のように個人を氏族名で呼ぶ感覚。
