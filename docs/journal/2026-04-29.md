## 発生した問題と対応

### 問題 1: SSH セッションが Console session に届かない

**症状**: `Start-Process https://...` を SSH 経由で実行しても Chrome に新タブが開かない
**原因**: OpenSSH for Windows のセッション分離（SSH session ≠ Console session 1）
**対応**: ユーザー側で Chrome 手動起動 → bat ファイル経由で 4 URL 一括オープン
**学び**: SSH からの GUI 操作は本質的に困難。ファイル DL や config 編集は SSH で、GUI 操作はユーザー側で、と役割分担

### 問題 2: SuperWhisper Onboarding が設定をリセット

**症状**: SSH で preferences.json を編集して onboarding スキップ + showExperimentalModels=true 設定するも、SuperWhisper 起動時に巻き戻される
**原因**: SW 起動時にバリデーション/サニタイズが走り、不整合な設定を初期値に戻す
**対応**: GUI で onboarding を素直に通過 → モード設定は `default.json` で永続化される（こちらは生き残る）
**学び**: 一部のアプリは preferences の SSH 編集を許さない、その場合は GUI 操作が必須

### 問題 3: SuperWhisper Windows v1.3.9 に Scribe が不可視

**症状**: Voice Model dropdown に「Scribe」「ElevenLabs」表示なし、`showExperimentalModels=true` でも変化なし
**原因**: **Scribe は SuperWhisper Pro 限定機能**（`licenseType: empty` で Free tier 確定）
**対応**: 課金せず Python 自前路線で Scribe 直叩き構成 (`keybind_scribe.py`)
**学び**: 勝間 4/27 18:17 ツイ「SuperWhisper 買い切り、Scribe 課金なし」は勝間さん**自身が Lifetime 購入済 + BYO ElevenLabs API**を意味していた

## 学んだこと

- **勝間 X の likes 動態**: 本人主張ツイは 24h で 5-10 倍化け、返信ツイは横ばい。監視時にこの 2 種を区別する構造が必須
- **漢数字 → 算用数字は LLM 不要**: 「単位が続く時のみ変換」ルールでローカル正規表現で 95% カバー、慣用句は単位なしで自動回避
- **SuperWhisper Free vs Pro**: Free 枠は S1-Voice / Ultra / Nova 等のみ、Scribe は Pro 限定（または Lifetime 購入 + BYO key）
- **Push-to-talk ホットキー設計**: 左 Alt は Alt+Tab/F4/メニューと干渉、無変換キーが日本語キーボードでは最適
- **OpenSSH for Windows セッション分離**: GUI 操作（ブラウザ起動、アプリ最前面化）は SSH 経由で不可、ファイル/config/プロセスは可
