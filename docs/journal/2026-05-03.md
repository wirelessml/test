### 推定原因 (確証なし)

**Joy-Con 2 (R) のバッテリー枯渇の可能性が最も高い**:
- 5/2 Phase 2 検証以降 (2 日間) 充電せず
- Joy-Con 2 は BLE でも 24h 以上で枯渇するレベルの低容量
- LED は薄っすら点いた (バッテリー残少警告 or wake up indicator) かもしれない
- 本格的な BLE radio 動作にはバッテリー残量 X% 以上必要、それを下回ると advertisement 出さなくなる挙動

代替仮説:
- Joy-Con 2 firmware で SYNC ボタンの長押し時間が変わった
- ユーザーが SYNC ボタンの位置を間違えていた
- Joy-Con 2 ハードウェア故障 (radio TX 部分)

### 次回再開時の手順

1. **Joy-Con 2 (R) を USB-C で 30 分以上充電** (Switch 2 ドック or 直接 USB-C ケーブル)
2. SYNC ボタンの位置確認: 側面レール (Switch 取付側) の小さい黒い凹みボタン
3. 5 秒長押し → LED 4 つが `1→2→3→4` と循環することを目視確認
4. Mac の BT 設定で advertisement 検知できるか確認 (一発診断)
5. 検知できれば Windows でも検知できる → ペアリング → `dotnet run` で Phase 3 検証

   - 設定ファイル `Mouse Mode.cfg` 自動生成も `C:\Users\wirel\Tools\JoyToKey\` ディレクトリ
     未存在で保存失敗

3. **Gopher360 v0.989 採用 (本命)**:
   - 設定 0 で動く 65 KB single exe、デフォルトでマウスマッピング済
   - 当初 URL を間違えて配布 (`v1.0.0/Gopher360.exe` → 正しくは `v0.989/Gopher.exe`)、
     WebFetch で GitHub Release API を見直して訂正
   - インストール先: `C:\Users\wirel\Tools\Gopher.exe`
   - 動作確認: コントローラの左スティックでマウスカーソルが動作

4. **タスクスケジューラで自動起動 + 管理者権限**:
   - `Register-ScheduledTask -TaskName "Gopher360 AutoStart" -RunLevel Highest`
   - スタートアップフォルダ方式 (ユーザー権限のみ) を先に削除して二重起動回避
   - 管理者権限で起動するため、タスクマネージャー等の admin アプリも操作可能

5. **VLC 3.0.23 を winget で導入**:
   - `winget install VideoLAN.VLC --accept-source-agreements --accept-package-agreements`
   - インストール先: `C:\Program Files\VideoLAN\VLC\`
   - 動画再生スタックを整理:
     - 4K UHD BD = PowerDVD 14
     - 通常 BD/DVD = PowerDVD 14 兼用
     - 汎用動画 (mp4/mkv/webm) = **VLC** 新規
     - YouTube DL = yt-dlp
     - エンコード = ffmpeg

### 朝の Joy-Con 2 とのコントラスト

朝 (Phase 3) のボツ要因と今回の成功要因の対比:

| 要素 | Joy-Con 2 (5/3 朝、ボツ) | 今回のサードパーティ (5/3 夕、成功) |
|---|---|---|
| 接続 | BLE (advertisement 出ず) | **有線 USB-C** (即認識) |
| プロトコル | カスタム HID (Joy-Con 2 専用) | **XInput 標準** (Microsoft 公式ドライバ) |
| Windows 認識 | デバイスツリーに出ず | `XBOX 360 Controller For Windows` と認識 |
| ドライバ | カスタム実装必要 (.NET BLE) | **不要** |
| ソフト | 自作 (Phase 1+2+3 scaffold) | **Gopher360 (既製品)** |
| 結果 | ハードウェア / バッテリー疑い → 断念 | 5 分で常駐マウス化完了 |

教訓: **マウス化ニーズなら最初から XInput 互換有線パッドを買うのが正解**。Joy-Con 2 は
BLE プロトコル独自仕様で、Mac でも Windows でも開発コストが見合わない。
