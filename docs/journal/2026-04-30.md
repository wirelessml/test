## 追補 (10:26〜11:15 JST) — Remote Control 接続失敗の RCA + claude-obsidian SessionStart hook 修正 + 上流バグ報告

### 経緯

10:26 の新セッション起動直後に `/remote-control` を実行 → 27.8 秒待たされた末に "Session creation failed" エラー。20 分後の 10:47 に再試行したら 0.8 秒で成功。なぜ 1 回目だけ失敗したかを systematic-debugging で根本原因まで追跡 → claude-obsidian の SessionStart prompt hook を削除して再発防止 → 11:10 までに上流 2 リポ (Anthropic / claude-obsidian 作者) の既存 issue にコメント追加で報告完了。

### 根本原因

claude-obsidian plugin v1.4.3 の `SessionStart:startup` hook が **prompt 型 hook を使用**しており、Claude Code 2.1.123 の内部バグを踏んでいた。SessionStart hook の `hook_non_blocking_error` attachment にエラーが記録されていた:

```
stderr: "Failed to run: ToolUseContext is required for prompt hooks. This is a bug."
exitCode: 1, durationMs: 1
hookName: SessionStart:startup
```

(`This is a bug.` は Claude Code 自身が「内部バグ」と認めている文言)

このエラー attachment が session の dirty state として残った状態で `/remote-control` を実行したため、Anthropic backend の session creation API で 27.8 秒の internal timeout 発動。20 分後は session が定常状態になり attachment 0 個で 0.8 秒で通った。

### 1 回目失敗 vs 2 回目成功 の決定的差分

| 指標 | 1 回目 (10:26:46) | 2 回目 (10:47:18) |
|---|---|---|
| session 起動からの経過 | 8.2 秒 | 約 20 分 |
| 直前 attachment 数 | 8 個 (約 64KB の hook 出力 + error) | 0 個 |
| transcript 累積 | 50.3 KB | 51.3 KB (+1KB のみ) |
| 結果 | 27.8 秒後に Session creation failed | 0.8 秒で成功 |

ペイロードサイズではなく **dirty hook state の有無**が決定要因。

### 対応 (両ファイル編集)

claude-obsidian の hooks.json から `SessionStart` 配列の **2 つ目 (`type: "prompt"`)** だけを削除。1 つ目の command 型 hook (`cat wiki/hot.md`) は機能上同じこと (hot.md の内容を context に注入) をするので残す → **機能損失ゼロで bug 回避**。

編集対象 (2 ファイル、cache と marketplace):
- `~/.claude/plugins/cache/claude-obsidian-marketplace/claude-obsidian/1.4.3/hooks/hooks.json`
- `~/.claude/plugins/marketplaces/claude-obsidian-marketplace/hooks/hooks.json`

バックアップ: `*.json.bak.20260430-110352` (両ディレクトリ)

検証 (verification-before-completion 準拠):
- ✓ JSON syntax (両ファイル `jq empty` OK)
- ✓ SessionStart 配列が 2 → 1 (command 型のみ残存)
- ✓ 他 hook (PostCompact / PostToolUse / Stop) 無傷
- ✓ cache ↔ marketplace 完全一致 (ドリフトなし)
- ✓ diff で削除したのは `type: "prompt"` の 4 行のみ

### 学んだこと

- **エラーメッセージの末尾 "This is a bug." を見逃さない**: Claude Code 内部バグ。Anthropic にレポート価値あり。
- **dirty state を含んだ session を export する API は危険**: Remote Control の session creation のような「session state を backend に送る」操作は、起動直後の不安定状態 (hook 完了直後) を避ける運用にする。応急対応としては「セッション起動から 30〜60 秒待ってから `/rc`」。
- **冗長な hook ペアは整理候補**: 同じ目的の hook が command + prompt の 2 種で重複していた。プラグイン側の冗長設計に気づけたのは bug のおかげ。

### 確定した含意

1. **iiyama STYLE Infinity 2018 BTO は「Pioneer BDR-211JBK + PowerDVD 14 UHDBD-OEM」のセット出荷だった** — 当時 ¥30-50K 相当の追加オプション付き、上位ワークステーション級の構成

### macOS キーボード問題 (未解決、再起動後に再評価)

- Karabiner Virtual Keyboard を ANSI から JIS に変更しようとしたが挙動変わらず
- 調査の結果、**macOS の物理キーボードタイプ判定は全部正しく**設定されていた:

  ```
  4096-1539-0 = 42  (内蔵キーボード = JIS ✓)
  79-1226-0   = 42  (LITEON UCL111UBK1 = JIS ✓)
  9-49396-0   = 40  (HCT 片手キーパッド = ANSI ✓)
  ```

  (42 = JIS、40 = ANSI)

- **真の原因**: 入力ソースに「ABC」(米国 ANSI レイアウト) が含まれてて、それが選択されてる時に Shift+2 = `@` になる (JIS なら `"`)
- 解決法: システム設定 → キーボード → 入力ソース「編集...」から **ABC を削除** or **日本語 IME 側に切り替え** (Shift+2 = `"` の方が JIS、`@` の方が ABC)
- **再起動後にメモ.app で挙動再確認 → ABC を削除する流れ**

### Fusion connector 仕様 (実機検証で判明したこと)

| 項目 | 値 |
|---|---|
| connector 名 | Autodesk Fusion |
| 説明 | Create, modify, and inspect CAD geometry in Fusion |

| ライセンス | Personal Use Free で OK (有償 subscription 不要) |
| 通信方式 | localhost-only (MCP server は本機内、データ外部送信なし) |
| OS | Windows / Mac 両対応 (検証は Windows 11 25H2) |

### 学んだこと

- **macOS Tahoe の入力ソース API は厳格化された**: `TISSelectInputSource` が paramErr で programmatic 切替不可、defaults plist 直書きも Selected フィールドが OS 側に上書きされる。3rd party IME (Google IME 3.33) はそもそも System Settings ピッカーに出ない。当面 Apple ことえりで戦う。
- **Anthropic Creative Connectors (4/28 ローンチ 9 個) は Claude Desktop 内蔵ディレクトリ経由専用**: MCP Registry API には未登録、Claude.ai web では「Claude for Desktop が必要」と表示される、Claude Code CLI でも `mcp add` 用 URL は公開されてない。今後 vendor 提供 remote MCP として Claude.ai にも降りてくる可能性はある。
- **M1 8GB で Claude Desktop + Fusion + Claude Code の同時稼働はメモリ的に厳しい** (試算 ~9.4-10 GB、swap 確実)。しゅん先生 PC (i7-8700K + 16GB DDR4 + Acer FA100 NVMe) は十分余裕、適材適所で完全動作。
- **Desktop の git 管理は project structure とエフェメラルファイルが混在しがち**。`_archive_YYYY-MM-DD/` のフォルダで月次退避する運用が機能する (claude-md-diet スキルの bounded context 思想と整合)。
