### 14:45〜15:30 キーボード ASMR 制作（合成 → 実録音）

**フェーズ 1: numpy 合成版（60 秒）**:
- Poisson 過程の打鍵タイミング、Key/Space/Enter 別の周波数帯
- ステレオパン ±0.85 ランダム配置
- 195 キー打鍵 / 3.2 keys/sec / 10MB ステレオ WAV
- → ユーザー判定: 「キーボードらしくない」

**フェーズ 2: 動画版（PIL + ffmpeg）**:
- 1080×1080 / 30fps / SSD 記事冒頭をタイプライター風表示
- 1,800 フレーム生成 → 1.8MB H.264 mp4

**フェーズ 3: pdf-reader Next.js でブラウザ閲覧公開**:
- `app/asmr/page.tsx` 新規作成
- `public/keyboard-asmr.{mp4,wav}` 配置
- LAN 公開: http://192.168.2.160:3000/asmr

**フェーズ 4: 実音録音（最初は無音）**:
- ffmpeg avfoundation `:3`（MacBook Air 内蔵マイク）
- 60 秒録音 + 中断後 10 分録音 → どちらも完全無音（max abs 0）
- **原因**: macOS Terminal にマイク権限がなかった
- ユーザーが System Settings で権限付与 → 再録音

**フェーズ 5: 実音録音 8:12 + ピーク同期動画**:
- 録音: 492.38 秒 mono 44.1kHz、max 9.5%、RMS 49、Nonzero 98.9%
- ピーク検出: scipy.signal.find_peaks、95 パーセンタイル閾値
  - 最初の試行は閾値 0.015 floor が高すぎて 18 ピークしか検出できず
  - 中央値 ×4 fallback も録音が静かで誤動作（0.0030 で 176 ピーク）
  - 最終的に **percentile(envelope, 95)** で **2,158 ピーク（4.38 keys/sec）** ← 自然な打鍵密度
- 動画: 14,771 フレーム生成、Substack 記事ドラフト全文 7,407 字を表示テキストに使用、22 行スクロール
- 出力: 13MB H.264 mp4
- `/asmr` ページに **実録音版 + 合成版（比較用）** の 2 段表示で公開

## 重要な学び（ナレッジ保存）

### 1. opendataloader-pdf は政府文書も学テも大量処理できる

- 256 ページ白書を 38 秒（0.15 秒/ページ）
- 縦書き日本語混在でも構造化可能
- 唯一の罠: Java 11+ 必須、Mac の Homebrew formula は PATH 設定が必要
- 仲さんの用途（学テ分析、政府白書分析、Substack 記事ドラフト）に完全フィット

### 2. voice-pro は Mac で時間を溶かす罠

- README に "Mac 未検証" と明示
- Miniconda が x86_64 のみ提供 → Rosetta 2 経由
- `openai-whisper` ビルドで `pkg_resources` エラー
- 4.9GB の installer_files が無駄になる
- **代替**: Kokoro TTS（Mac ネイティブ、軽量、用途に合致）

### 3. Kokoro TTS は M1 ネイティブで圧倒的に軽い

- 82M パラメータ、MLX ゼロコピー
- Claude Code の Stop フックと組み合わせると応答が音声化
- `CLAUDE_VOICE=1` ガードでオプトインなので既存セッションを汚染しない
- 設定: ~/.claude/hooks/kokoro-tts/ + settings.json Stop hook

### 4. macOS マイク録音は Terminal 権限を確認すること

- 権限なしでも ffmpeg はエラーを出さず無音 (全サンプル 0) を書き出す
- 検証法: 録音後に `np.max(np.abs(data))` で 0 でないことを確認
- System Settings → Privacy & Security → Microphone で Terminal を ON

### 5. 打鍵ピーク検出の閾値はパーセンタイルで決める

- 中央値 × N の絶対値 floor は録音音量によって誤動作する
- `np.percentile(envelope, 95)` のような相対指標が頑健
- 自然なタイピング密度は 4-5 keys/sec、これに合うように調整
