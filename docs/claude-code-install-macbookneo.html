この手順は人間が手動で実行することも、AI エージェント（Claude Desktop Code mode / Codex Computer Use / Manus My Computer）に依頼して自動実行させることも可能です。エージェントに依頼する場合は次節 §0 のプロンプト例を使用。

0. AI エージェントへの依頼（新規 Mac セットアップ時）

MacBookNEO で Claude Desktop / Codex / Manus のいずれかを先に起動して本人がサインイン済みなら、下記プロンプトをそのまま貼ってインストールを一任できる。

0.1 共通プロンプト（どのエージェントでも使える）

https://wirelessml.github.io/test/docs/claude-code-install-macbookneo.html
の手順に従って、この Mac に Claude Code CLI を**スタンドアロン版で**インストールしてください。

必須ルール:
1. npm 版は絶対に入れない（§2.3）。
2. パスは ~/.local/bin/claude、データは ~/.local/share/claude に置く（§3.1）。
3. OAuth 認証は私が手動でブラウザに入力するので、claude コマンド初回起動までやったら待機してください（§4.1〜4.2）。
4. `~/.claude/settings.json` は §5.1 の現行内容をそのまま書き込んでください（`CLAUDE_CODE_DISABLE_1M_CONTEXT` は削除したまま）。
5. インストール後に `claude --version` と `claude doctor` の出力を私に見せてください。

途中で権限ダイアログが出たら止めて私に確認してください。シェル操作は可能な限り最小限の承認範囲で。

0.2 エージェント別の補足

  • Manus（My Computer）: 自律実行が最も得意。上記プロンプトをそのまま投げれば、Terminal 起動 → curl 実行 → 認証前ステップまで完走する想定。
  • Codex（Computer Use）: Terminal.app を事前に「常に許可するアプリ」へ追加しておくと都度承認ダイアログが減る。
  • Claude Desktop Code mode: ローカル Mac を操作する場合は SSH 越しが基本のため、自機を localhost SSH 先として登録するか、代わりにメイン会話でコマンドを出してもらって手動コピペする運用が無難。

0.3 エージェント実行時の最小コマンド列（監視用）

エージェントが何を打っているか確認したいときの対応。下記5行で認証直前まで到達する。

curl -fsSL https://claude.ai/install.sh | bash
export PATH="$HOME/.local/bin:$PATH"
claude --version
mkdir -p ~/.claude
# settings.json は §5.1 から転記

認証以降（§4.2）は人間が手動で行う前提。

1. 概要

項目値
インストール方式スタンドアロン版（native）推奨
バイナリ場所~/.local/bin/claude → ~/.local/share/claude/versions/<VERSION>
設定~/.claude/（共通） / ~/.claude.json（プロジェクト別）
認証Claude Max OAuth（Keychain保存）or ANTHROPIC_API_KEY
現行版2.1.112（2026-04-17）

選定根拠: なぜスタンドアロン版か

npm版は避ける。 @anthropic-ai/claude-code をグローバルインストールすると、Node.js 更新・npm cache 操作で壊れる。Windows PC（MASU-P55）で npm版とスタンドアロン版が競合した事故実績あり（2026-04-14 対応済）。
  • 単一バイナリ（約200MB）、依存なし
  • CLI内蔵アップデーターが新版をバックグラウンドDL → 次回起動で symlink 差し替え
  • アンインストールは rm -rf ~/.local/share/claude ~/.local/bin/claude ~/.claude ~/.claude.json だけ

2. 前提条件

必須

sw_vers -productVersion   # 26.5（12以上で動作）
uname -m                   # arm64（Apple Silicon）
echo $SHELL                # /bin/zsh

推奨（便利ツール）

  • Homebrew 5.x 任意 brew --version
  • Node.js 20+ 任意 MCP サーバー用、現環境 v22.15.0
  • Git 2.x 任意 Claude Code が自動で git 状態を読む
  • gh CLI 2.x 任意 GitHub操作の補助

PATH 確認

echo $PATH | tr ':' '\n' | grep -q "$HOME/.local/bin" && echo OK || echo MISSING

無い場合は ~/.zshrc に追記:

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
絶対に避ける:
  • sudo npm install -g @anthropic-ai/claude-code
  • /usr/local/bin へのコピー（更新不能になる）

1M context 有効化: CLAUDE_CODE_DISABLE_1M_CONTEXT を 書かないことで既定の 1M が効く。2026-04-17 に既存設定から削除済み。環境変数は起動時に読まれるため、反映は次セッションから。

9. アップデート

  • autoUpdates: false でも CLI 内部で新版チェック → バックグラウンドDL
  • 新版は ~/.local/share/claude/versions/X.Y.Z に配置
  • 次回起動時に symlink が最新へ差し替わる
  • 過去2〜3世代は残存、以前は自動削除
# 現行版
claude --version

# 全バージョン
ls ~/.local/share/claude/versions/

# 強制更新
curl -fsSL https://claude.ai/install.sh | bash

# ダウングレード（symlink を手動で張り替え）
ln -sf ~/.local/share/claude/versions/2.1.110 ~/.local/bin/claude

10. トラブルシュート

症状 claude: command not found

PATH 漏れ。export PATH="$HOME/.local/bin:$PATH" を ~/.zshrc に追加 → source ~/.zshrc

症状 OAuth 認証が通らない

  • Keychain ロック: security unlock-keychain ~/Library/Keychains/login.keychain-db
  • トークン破損: rm ~/.claude/.credentials.json → 再 /login

症状 起動時 Bun クラッシュ（Segmentation fault）

原因: メモリ不足（M1 8GB で他アプリ多い）

  • OBS / Claude Desktop / Chrome を終了
  • Claude Desktop は常用しない（4/14 で停止済）

症状 1M context が効かない

~/.claude/settings.json の env から CLAUDE_CODE_DISABLE_1M_CONTEXT を削除。環境変数は起動時読み込みのため新セッションから有効。

症状 MCP 接続失敗

/mcp                  # 状態確認
claude --mcp-debug    # デバッグ起動

症状 "Usage Limit Reached"

  • Max プランの週次/月次リミット到達
  • 対処: 待つ（週次は木曜リセット）/ /model sonnet へ切替 / /model opusplan 採用

症状 セッションが固まる

ps aux | grep -i claude | grep -v grep
pkill -f "claude$"

12. アンインストール

完全削除

# バイナリ
rm -rf ~/.local/share/claude
rm ~/.local/bin/claude

# 設定・セッション
rm -rf ~/.claude
rm ~/.claude.json

# Keychain
security delete-generic-password -s "Claude Code-credentials"

設定だけリセット

cp ~/.claude/settings.json ~/.claude/settings.json.bak
cp ~/.claude.json ~/.claude.json.bak
rm ~/.claude/settings.json ~/.claude.json
