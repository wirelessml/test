# Qwenファミリーガイド（Alibaba Cloud）

## モデル一覧

| モデル名 | 用途 | 説明 |
|---------|------|------|
| **Qwen 2.5** | 考える（LLM） | ChatGPTのようなテキスト生成・会話・推論 |
| **Qwen 3.6** | 考える（LLM・最新） | 27B Denseモデル、ローカルLLMの新覇者 |
| **Qwen3-TTS** | 喋る（音声合成） | テキストを声にする。声クローンも可能 |
| **Qwen-VL** | 見る（画像理解） | 画像を理解する視覚言語モデル |

| **Qwen-Coder** | コードを書く | プログラミング特化 |

## Qwen 3.6 詳細（2026年4月時点）

### スペック
- **27B Dense**: 主力モデル。MoEではなくDense（密）モデル
- **35B-A3B MoE**: 16GB環境向け。賢さを維持しつつ省メモリ
- 推論の安定感、ストーリーテリング、翻訳で高品質
- 新手法「DFlash」で3〜5倍の速度向上の可能性

### 動作環境（GPU推論 = 実用速度）

| GPU（VRAM） | 推奨モデル | 速度 |
|------------|----------|------|
| RTX 3090/4090（24GB） | 27B Dense Q4 | 10〜50トークン/秒（実用的） |
| RTX 3060（12GB） | 小型モデルのみ | やや遅い |
| Apple Silicon 32GB+ | 27B Dense | メモリ共有で動作可能 |
| Intel UHD等（VRAM 128MB） | GPU推論不可 | — |

### CPU推論（グラボなし）の現実
- CPUだけでもllama.cpp等で動作はする
- ただし**1〜5トークン/秒**（1文返すのに数十秒〜数分）
- ライト操作「電気つけて」程度なら使えるが、会話は非実用的

### 手持ち環境での可能性

| 環境 | GPU | RAM | 27B動作 | 実用性 |
|------|-----|-----|---------|--------|
| M1 MacBook Air | 統合GPU（8GB共有） | 8GB | **不可** | メモリ不足 |
| デスクトップPC (i7-8700K) | Intel UHD 630 (128MB) | 16GB | **CPU推論のみ** | **激遅、非実用的** |
| しぶ M5 MacBook Air | 統合GPU（32GB共有） | 32GB | **動作可能** | Apple Siliconのメモリ共有で実用的 |
| RTX 3090/4090搭載PC | VRAM 24GB | — | **最適** | 最速 |

## ローカルLLM活用シナリオ

### 実現可能なパターン
```
RTX 3090/4090搭載PC + Ollama + Qwen 3.6 27B
→ OpenClaw経由でSwitchBot AIハブに接続
→ 完全無料の高速AIスマートホーム
```

```
Apple Silicon 32GB Mac + Ollama + Qwen 3.6 27B
→ しぶのM5 MacBook Airなら実用的に動く
→ 「Claude動かしっぱなし用」に加えてローカルLLMも
```

### 現環境（M1 8GB / Intel UHD 16GB）の結論
- ローカルLLMは非実用的
- **クラウドAPI（Claude Pro / GPT）が現実解**
- SwitchBotのAI機能を使うならAPI課金（月数百円程度）が最もコスパ良い

## 過去の試行結果（M1 8GB）
- Qwen3-TTS: 動作したが声クローンの精度が低い（似てなかった）
- RVC学習: DataLoaderでハング（CPU/MPS両方）、M1 8GBでは不可能
- 結論: M1 8GBではQwen系の大型モデルは厳しい。ElevenLabs（クラウド）が現実解
