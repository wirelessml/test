Maestri 使い方ガイド

複数AIエージェントを無限キャンバスで操る macOS ターミナルアプリ

Maestriとは

Maestriは、Claude Code・Codex・OpenCodeなどのAIコーディングエージェントを1つの無限キャンバス上に並べて同時に操作できるmacOSネイティブアプリ。Swift + Metal製で軽量、エージェント間の接続・対話も可能。

インストール

1

公式サイトからDMGをダウンロード

2

DMGを開き、Maestri.appを /Applications にドラッグ

3

初回起動時にGatekeeper警告が出たら「開く」を選択
または xattr -cr /Applications/Maestri.app で解除

ターミナルの作成

キャンバス上にAIエージェントのターミナルを配置する方法。

1

上部ツールバーの ターミナル アイコンをクリック（ショートカット: T）

2

キャンバス上の好きな場所をクリック

3

「新規ターミナル」ダイアログでエージェントを選択

  • Claude Code — コマンド: claude
  • Codex — OpenAIのコーディングエージェント
  • Gemini CLI — GoogleのAIエージェント
  • OpenCode — オープンソースエージェント
  • Shell — 通常のシェル
4

「作成」をクリック → キャンバスにターミナルが出現

ターミナル間の接続

2つのターミナルを繋ぐと、エージェント同士が直接対話できるようになる。

接続方法

方法1: サブツールバー
  1. 接続元のターミナルのタイトルバーをクリックして選択
  2. サブツールバー（上部中央に出現）の Y字アイコン（「ターミナルに接続」）をクリック
  3. カーソルに線がついてくる → 接続先のターミナルをクリック
  4. 物理アニメーション付きのケーブルで接続完了
方法2: キーボードショートカット

ターミナルをカード選択状態（内部フォーカスなし）で L を押す → 接続先をクリック

※ターミナル内部にフォーカスがあると L が入力されてしまうので注意

接続で何ができるか

  • Maestri Agent Skill が各ターミナルに自動インストールされる
  • エージェント間でプロンプトの送受信が可能
  • エージェント非依存 — Claude Code ↔ Codex ↔ OpenCode どの組み合わせもOK
  • 1つのターミナルに複数の接続を持てる

使い方の例

「Claude Code #2に、shibu-live.pyのコードレビューを依頼して」

→ エージェントが自動で隣のエージェントにプロンプトを送信・回答を受信

スキルが自動読み込みされない場合: 「maestriを使って[エージェント名]に聞いて」と指示する。

ノートとの接続

ターミナルをノート（付箋）に接続することも可能。接続すると、エージェントがノートの読み書きをMaestri CLI経由で行える。セッションをまたいで情報を残す共有メモとして機能する。

ノート同士をチェーンやツリー状に接続でき、エントリーポイントのノートだけターミナルに繋げば、エージェントが全体を辿れる。

エージェントロール

各ターミナルに役割を割り当てて、自動的にエージェントの振る舞いを制御する。

ロールの例
ロール役割
Lead全体を統括し、タスクを他のエージェントに委任
Coder実装に集中
Reviewerコードレビューと指摘
Testerテスト作成と実行

設定: Settings → Agents でロールの作成・編集。各ロールには名前・カラーバッジ・指示セットを設定。ロールはサブディレクトリの CLAUDE.md や AGENTS.md を通じてエージェントに注入される。

チーム開発時は .maestri を .gitignore に追加すること（ロール設定は現在共有不可）

ワークスペースとFloors

ワークスペース

プロジェクトごとに別のキャンバスを作成。tmux風ショートカットで瞬時に切り替え可能。

  • Free 1ワークスペース
  • Pro $18 ワークスペース無制限 + Mac 2台

Floors

ワークスペースの分離コピーをAPFS copy-on-writeで即座に作成。ディスク使用量は最小限。別ブランチの並行作業を干渉なしで実行し、後でマージまたは破棄できる。

Ombro（オンデバイスAI）

Apple Foundation Models搭載のアシスタント。エージェントの進捗を監視し、タスク完了時に要約と次のステップを提案する。アプリ外のフローティングウィンドウで表示。

  • API不要、クラウド送信なし — 完全ローカル処理
  • メニュー: ファイル → Ombro

M1 8GBでの運用Tips

Apple Silicon 8GBメモリではMaestri + OBS同時運用は高負荷になる。以下を推奨:

  • Chromeなどの重いアプリは閉じる（Chrome終了でload 18→2に改善した実績あり）
  • Claude Codeは1セッション約26% CPU / 340MB RAM消費
  • Maestri自体は約27-30% CPU
  • OBS配信(50%) + Maestri(30%) + Claude Code 2台(50%) = load 5〜7程度
  • 3台以上のClaude Codeは16GB以上のMacを推奨
